Keyphrase Extraction and Generation Datasets

Loading datasets

This how to guide looks at loading different datasets for training and evaluating keyphrase extraction and generation models. The library intends to provide support for loading both publicly available datasets as well as privately held datasets. Current version only supports loading publicly available datasets. See below for a full list of publicly available datasets.

Loading data for training and evaluating keyphrase extraction models

Load publicly available data from Huggingface Hub for keyphrase extraction

from transformerkp.data.dataset_loaders import Inspec

inspec_data_ke = Inspec(mode="extraction").load()
print(inspec_data_ke.train)
print(inspec_data_ke.test)
print(inspec_data_ke.validation)

The document field of the dataset contains the whitespace tokenized document and the doc_bio_tags column of the dataset contains their corresponding tags in the B-I-O scheme. 

# iterate over the dataset

Load your own dataset for keyphrase extraction

One can also load custom dataset for keyphrase extraction. It is expected that the dataset will have the document and doc_bio_tags columns. Since, we treat keyphrase extraction as a sequence tagging task we expect the availability of the input document with tokens tagged with the B-I-O scheme. The document field of the dataset contains the whitespace tokenized document and the doc_bio_tags column of the dataset contains their corresponding tags in the B-I-O scheme. 

ke_dataset = KeyphraseExtractionDataset(
    train_file="{path_to_train_data_file}",
    validation_file="{path_to_validation_data_file}",
    test_file="{path_to_test_data_file}",
).load()

You can also mention the specific splits to be loaded

ke_dataset = KeyphraseExtractionDataset(
    train_file="{path_to_train_data_file}",
    validation_file="{path_to_validation_data_file}",
    test_file="{path_to_test_data_file}",
    splits=["train", "test"]
).load()

At this moment, only json and csv formats are allowed for custom datasets. Look at the json and csv samples provided in the examples/data/keyphrase/sample_data. In order to know more about the task of keyphrase extraction and the format of the dataset feel free to look at A Tutorial on Identifying Keyphrases from Text.

Load your own dataset for keyphrase generation

One can also load custom dataset for keyphrase generation. It is expected that the dataset will have the document and at-least one of extractive_keyphrases or abstractive_keyphrases or keyphrases columns.

kg_dataset = KeyphraseGenerationDataset(
    train_file="{path_to_train_data_file}",
    validation_file="{path_to_validation_data_file}",
    test_file="{path_to_test_data_file}",
).load()

You can also mention the specific splits to be loaded

kg_dataset = KeyphraseGenerationDataset(
    train_file="{path_to_train_data_file}",
    validation_file="{path_to_validation_data_file}",
    test_file="{path_to_test_data_file}",
    splits=["train", "test"]
).load()

At this moment, only json and csv formats are allowed for custom datasets. Look at the json and csv samples provided in the examples/data/keyphrase/sample_data.

In order to know more about the task of keyphrase generation and the format of the dataset feel free to look at A Tutorial on Identifying Keyphrases from Text.

Saving

You can save the loaded data splits in any of the three formats json, parquet and csv.

Saving in JSON Format

# save the dataset in a json file
inspec_data_ke.test.to_json("./inspec_test_ke.json")

Saving in Parquet format

# save the dataset in parquet
inspec_data_ke.test.to_parquet("./inspec_test_ke.parquet")

Saving in CSV format

# save the dataset in csv
inspec_data_ke.test.to_csv("./inspec_test_ke.csv")
